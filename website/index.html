The problem

      

        Production prompts outgrow their containers.
      


      

        

          In production agentic systems, prompts accumulate quickly. Instructions, persona
          definitions, tool descriptions, behavioral guidelines, conditional logic — they all
          pile into a single string buried inside Python or TypeScript files. Reviewing a prompt
          means reading past API client setup. A diff that changes two lines of prompt text is
          invisible next to surrounding application code.
        

        

          There is no mechanism for reuse. If two agents share a persona, that text gets copied.
          When it changes, it needs to change in every place. Non-engineers who own prompt content
          can't contribute without touching code. There is no format, no tooling, no discipline.
        

        

          PCL addresses this directly. .pcl files are first-class artifacts —
          structured, modular, and designed to live in version control alongside the rest of
          your codebase.

What PCL does

        

          Everything prompts need. Nothing they don't.
        

        

          Six focused capabilities. No runtime dependencies. Compiles to a plain string
          you drop into whatever API you're calling.
        

      


      


        
        

          

            
          

          
Modular composition

          

            Break prompts into named @blocks. Reference them anywhere with
            @include. Reuse across multiple agent configurations without copying.
          

        


        
        

          

            
          

          
File imports

          

            @import ./shared/persona.pcl brings another file into scope.
            Build a shared library of personas, tool descriptions, and guidelines that
            multiple agents consume.
          

        


        
        

          

            
          

          
Conditional logic

          

            @if variable: includes content only when a variable is truthy.
            @if not variable: for the inverse. Conditionals nest.
            No comparison operators — stays simple by design.
          

        


        
        

          

            
          

          
Variable interpolation

          

            ${variable} is replaced at render time from a dictionary you provide.
            Add a fallback with ${variable | default value}.
            Undefined variables with no default are a compile error.
          

        


        
        

          

            
          

          
Raw passthrough

          

            @raw / @end passes content through completely
            unmodified — no interpolation, no directive processing. Ideal for embedded
            JSON, code examples, or anything containing PCL special characters.
          

        


        
        

          

            
          

          
Plain text output

          

            The compiled output is a single string. No PCL runtime in your application —
            just pass the string to your LLM API. Works with OpenAI, Anthropic, Gemini,
            local models, or any API that takes a system prompt.
          

        


      

    

  



  


  
  

    


      

        
See it in action

        

          A complete PCL file.
        

        

          A research assistant system prompt — demonstrating every major language feature
          in one coherent file.
        

      


      
      


        
        

          
---1version: 1.2description: Research assistant system prompt---@import ./persona.pcl@import ./tools.pcl as tools2# Reusable date fragment@block date_notice:3    Today's date is ${date}.# A block composed from other blocks@block agent_setup:4    @include persona    @include date_notice    @include tools.search    @include tools.browse# Main prompt bodyYou are a research assistant.@include agent_setup@if premium:5    You also have access to the premium document index.@if not premium:    Upgrade to unlock the premium document index.# Embed a literal JSON example without triggering interpolation@raw6    When calling a tool, use this format:    {"tool": "${tool_name}", "args": {}}@endThe user's query is: ${query | no query provided}7

        


        
        

          


            

              1
              

                Frontmatter
                
YAML metadata between --- delimiters. Available to callers via .metadata at compile time — never appears in compiled output.

              

            


            

              2
              

                Import with alias
                
@import ./file.pcl as ns brings another file into scope under a namespace. Without as, the filename stem is used.

              

            


            

              3
              

                Block definition
                
@block name: defines a named fragment. Body is indented 4 spaces. Blocks emit no output on their own — only when @included.

              

            


            

              4
              

                Composed block
                
A block body can @include other blocks — even from imported files. Complex prompts compose from focused, reusable pieces.

              

            


            

              5
              

                Conditional logic
                
@if variable: includes the indented body only when truthy. @if not for the inverse. Conditionals can nest.

              

            


            

              6
              

                Raw block
                
@raw / @end passes content through completely unmodified — no interpolation, no directive processing. Use for JSON, code, or anything with PCL special characters.

              

            


            

              7
              

                Variable with default
                
${var | fallback} uses the fallback if var is undefined at render time. Without a default, an undefined variable is a compile error.

Getting started

        

          Up and running in two minutes.
        

        

          Install the package, write a .pcl file, compile it — then pass
          the output to your LLM API like any other string.
        

      


      


        
        

          

            1
            Install
          

          

            $ pip install pcl-lang
          

        


        
        

          

            2
            Write a .pcl file
          

          
# agent.pcl@block persona:    You are a helpful assistant.@include personaThe user asked: ${query}

        


        
        

          

            3
            Compile and render
          

          

            
# validate without output

            
$ pcl check agent.pcl

            
# compile and print

            
$ pcl compile agent.pcl

            
# render with variables

            
$ pcl render agent.pcl --var query="What is alignment?"

          

        


        
        

          

            4
            Or use the Python API
          

          

            
# from your application code

            
from pcl import render

            
prompt = render(

            
    "./agent.pcl",

            
    variables={"query": "What is alignment?"}

            
)

            
# pass `prompt` to your LLM API
